# SyncLife 홈페이지 스타일 가이드

> 컨셉: **테크 시안 · 클린/미니멀**. B2B 엔터프라이즈 마케팅 사이트.
> 이 문서는 모든 목업 HTML 작성/수정 시 **반드시 준수**한다.

## 1. 컬러
| 역할 | 값 | 용도 |
|------|----|----|
| Primary (sky) | `#0EA5E9` | 주요 버튼, 링크, 강조 |
| Primary-dark | `#0284C7` | hover, 진한 강조 |
| Accent (indigo) | `#6366F1` | 그라데이션 보조, 포인트 |
| Gradient | `linear-gradient(135deg,#0EA5E9,#6366F1)` | 히어로/배너 강조 |
| Ink (제목) | `#0F172A` | 본문 제목 |
| Text | `#334155` | 본문 |
| Muted | `#64748B` | 보조 텍스트 |
| Border | `#E2E8F0` | 구분선/카드 테두리 |
| BG | `#FFFFFF` | 기본 배경 |
| BG-soft | `#F8FAFC` | 섹션 교차 배경 |
| Success | `#10B981` / Danger `#EF4444` | 폼 피드백 |

## 2. 타이포
- 폰트: **Pretendard**(CDN) → fallback `system-ui, -apple-system, "Malgun Gothic", sans-serif`
- 스케일: Hero h1 `clamp(2rem,4vw,3.25rem)/700` · 섹션 h2 `clamp(1.6rem,2.6vw,2.25rem)/700` · 카드 h3 `1.25rem/600` · 본문 `1rem/1.7` · 보조 `0.9rem`
- 자간: 제목 `-0.02em`. 한글 가독성 우선.

## 3. 레이아웃
- 컨테이너 `max-width:1200px; margin:0 auto; padding:0 24px`
- 섹션 세로 패딩 `padding:96px 0`(모바일 `64px 0`) — **넓은 여백(미니멀)**
- 그리드: 카드 3열(`repeat(3,1fr)`, gap 24px) → 모바일 1열
- 라운드 `12px`(카드), `8px`(버튼/인풋), 큰 카드 `16px`
- 그림자: 절제 사용 `0 1px 3px rgba(15,23,42,.06)`, hover 시 `0 8px 24px rgba(15,23,42,.08)`

## 4. 컴포넌트
- **버튼**
  - Primary: 배경 sky, 흰 글씨, `padding:12px 24px; radius:8px; font-weight:600`
  - Ghost/Outline: 투명 배경 + sky 보더 + sky 글씨
  - hover: 살짝 어둡게 + `translateY(-1px)`
- **헤더(GNB)**: 흰 배경 sticky, 하단 1px 보더, 로고 좌측 + 메뉴 중앙/우측 + "문의하기"(Primary 버튼) 우측 고정. 솔루션은 드롭다운(WMS/OMS/AI). 모바일 햄버거.
- **푸터**: 진한 ink 배경(`#0F172A`) + 밝은 글씨, 회사정보/약관/소개서 다운로드/카피라이트.
- **카드**: 흰 배경 + 1px 보더 + 라운드 12px + hover 상승. 상단에 라인/소형 아이콘(SVG 단색).
- **히어로**: 좌측 카피 + 우측 비주얼(SVG 일러스트/추상도형). 배경은 흰색 + 은은한 그라데이션 블롭.
- **수치(stat)**: 큰 숫자(sky/gradient) + 라벨. 4열.
- **폼**: 라벨 상단, 인풋 `border:1px #E2E8F0; radius:8px; padding:12px`. 포커스 시 sky 보더(은은). 에러는 danger 텍스트 인라인.

## 5. 모션/인터랙션
- 미세하게: hover 트랜지션 `.2s ease`, 등장 애니메이션은 과하지 않게(옵션).
- 클린 컨셉 → 불필요한 그림자/네온/과한 애니메이션 금지.

## 6. 금지 사항
- 과한 유채색 남발(포인트는 sky/indigo만), 무지개색 금지
- 카드 위 카드(이중 래퍼), 행 배경색 떡칠 금지
- 네이티브 select/date 위젯 그대로 노출 → 디자인된 컴포넌트로(가능 범위)
- 좁은 여백/빽빽한 레이아웃 금지(미니멀 컨셉 유지)
- 이모지를 주요 아이콘으로 사용 금지(SVG 단색 아이콘 사용)

## 7. 자산/콘텐츠 표기
- 실데이터 미확보 항목(수치/로고/인증/연혁)은 **플레이스홀더**로 표기하고 `<!-- TODO: 실데이터 -->` 주석.
- 이미지 자산은 `mockup/assets/`에 두되 없으면 회색 placeholder 박스.

## 8. 반응형 브레이크포인트
- `>=1024px` 데스크탑(3열) · `768~1023` 태블릿(2열) · `<768` 모바일(1열, 햄버거)

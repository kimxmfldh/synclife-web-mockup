Requirements Guide

무엇을 어디까지 맞춰야 할까요?
도입 요구사항 정리부터

SyncLife는 검증된 표준 솔루션을 고객 업무에 맞춰 커스터마이징해 드립니다. 그 출발점은 "우리 업무에 무엇이 필요한지"를 정리하는 것입니다.

What to prepare

도입 요구사항: 무엇을, 어디까지 맞출지

수천만 원이 드는 차를 고민 없이 사는 분은 거의 없습니다. 옵션, 트림, 가격, 유지비를 따져 합리적으로 결정하죠. 우리 사업과 직결되는 시스템 도입도 마찬가지입니다.

SyncLife는 백지에서 만드는 외주 개발이 아니라, 이미 운영 중인 표준 WMS·CRM&OMS·AI를 고객 환경에 맞춰 커스터마이징해 라이선스로 제공합니다. 그래서 "어떤 솔루션을, 우리 업무에 맞게, 어디까지 손봐야 하는지"를 정리해 두면 제안과 도입이 훨씬 빨라집니다.

정리가 막막하셔도 괜찮습니다. 아래에서 ① 왜 정리가 필요한지 ② 어떻게 정리하는지 ③ 무엇을 점검하면 좋은지를 차례로 안내합니다.

Why it matters

요구사항 정리가 중요한 3가지 이유

첫째

제안·견적의 정확성

모호한 요구 = 모호한 견적. "그냥 WMS 도입하고 싶어요"와 "우리 입고 검수는 박스 단위, 채널은 3곳 연동, 정산은 화주별"처럼 정리된 요청은 전혀 다른 제안을 받습니다.

요구사항이 정리돼 있으면 표준 기능으로 충분한 부분과 커스터마이징이 필요한 부분이 분명해져, 근거 있는 견적과 합리적인 도입 범위가 나옵니다.

"그냥 WMS 도입" → 견적 들쭉날쭉
요구사항 정리 → 표준+커스터마이징 분리
둘째

말이 통하는 소통

고객은 '어떻게 업무를 더 잘 돌릴지'에 집중하고, 우리는 '표준 기능으로 어디까지 되고 무엇을 맞춰야 하는지'를 봅니다.

예를 들어 "입고 검수를 강화하고 싶어요"라고 하면 — 바코드 스캔 단위는 박스인가 낱개인가? 차이 발생 시 자동 보류인가? 표준 기능에 있는가, 커스터마이징인가? 이 관점 차이를 메우는 것이 요구사항 정리입니다.

"입고 검수 강화" 한 문장도
도입 관점에선 점검 항목이 여러 개
셋째

도입 결과가 기대와 일치

생각과 다른 결과만큼 당황스러운 일은 없습니다. 요구사항 없이 "알아서 맞춰주세요"로 넘기면, 우리에게 익숙한 표준 방식으로 설정되어 기대와 어긋날 수 있습니다.

"출고 취소가 돼야 한다"는 한 줄도 — 현장 담당자가 직접 취소하는지, 관리자 승인 후 취소인지에 따라 설정이 달라집니다. 예산이나 일정으로 범위를 조율할 때도 정리된 요구사항이 기준이 됩니다.

"출고 취소" → 현장? 관리자 승인?
정해두면 설정이 의도대로
How to organize

요구사항, 이렇게 정리하면 됩니다

전문 문서가 아니어도 괜찮습니다. 아래 3가지만 정리해도 충분합니다.

STEP 1

현재 업무 흐름 정리

지금 어떻게 일하고 있는지를 순서대로 적습니다. 누가(역할), 어디서(센터·채널), 무엇을(입고·출고·정산 등), 어떤 도구로(엑셀·타 시스템) 처리하는지. 가장 불편한 지점을 함께 표시하면 좋습니다.

STEP 2

표준 기능 대비 '꼭 필요한 것' 정리

우리 업무에서 반드시 있어야 하는 기능과 규칙을 적습니다. 예: "검수는 박스 단위", "정산은 화주별 월마감", "특정 권한만 단가 조회". 표준에 없으면 커스터마이징 대상이 됩니다.

STEP 3

연동·규모·인프라 정리

연동할 시스템(기존 ERP·쇼핑몰·택배사), 운영 규모(센터 수·사용자 수·월 물동량), 도입 인프라(클라우드/온프레미스) 희망을 적습니다. 도입 범위와 일정 산정의 기준이 됩니다.

Example

예시로 보는 요구사항 정리

막막하실 땐 아래 표처럼 한 줄에 요구사항 하나씩, 지금 방식(As-Is)과 원하는 모습(To-Be)을 함께 적어 보세요. 이 형식 그대로 엑셀에 옮겨 적으셔도 좋습니다.

요구사항 정의서 예시 — 물류·유통 도매업체
요구사항 ID 구분 요구사항명 현재 방식 (As-Is) 원하는 동작 (To-Be) · 상세 사용 역할 우선순위 비고 · 조건
REQ-001입고박스 단위 입고 검수 입고 시 송장과 실물을 눈으로 대조하고 엑셀에 수기로 기록 PDA로 박스 바코드를 스캔해 발주 수량과 자동 대조. 수량이 맞지 않으면 해당 입고 건을 검수 보류로 전환하고 관리자에게 알림 → 관리자 확인 후에만 입고 확정 입고 담당자 / 센터 관리자 필수 박스 단위(낱개 아님), 입고 시 유통기한 함께 입력
REQ-002출고출고 취소 승인 통제 출고 지시 후 취소는 전화로 요청, 시스템에 기록이 남지 않음 현장 담당자는 출고 취소를 요청만 하고, 센터 관리자가 승인해야 실제 취소. 취소 사유 필수 입력 + 취소 이력 보관 출고 담당자(요청) / 관리자(승인) 필수 이미 송장 출력된 건은 취소 불가 등 예외 정의 필요
REQ-003정산화주별 월마감 정산 월말에 엑셀로 화주별 입출고 건수를 집계해 수기 계산 화주별 보관료·입출고 작업료를 단가표 기준으로 자동 집계. 월 단위 마감 후 거래명세서·세금계산서 발행, 마감된 달은 데이터 수정 잠금 정산 담당자 / 화주(조회) 필수 단가 = 보관료(파렛트·일) + 입출고료(건당)
REQ-004연동쇼핑몰 주문 자동 수집 채널별 관리자 페이지에서 주문을 각각 내려받아 취합 카페24·스마트스토어·쿠팡 등 채널 주문을 주기적으로 자동 수집해 출고 대기열에 등록. 채널의 신규/취소/교환 상태도 동기화 출고 담당자 필수 연동 채널 목록·API 키(계정) 제공 필요
REQ-005권한단가·원가 조회 제한 모든 직원이 단가표 엑셀을 공유해서 봄 상품 단가·원가는 관리자 권한 계정만 조회. 일반 직원 화면에서는 금액 항목을 숨김 처리 관리자 / 일반 직원 선택 권한 등급별 노출 항목 정의 필요
REQ-006AI비정형 주문서 자동 변환 거래처가 메일·엑셀로 제각각 양식의 주문서를 보내면 수기로 재입력 다양한 양식의 주문서(메일 본문·엑셀·이미지)를 표준 주문 형식으로 자동 변환해 수주 등록. 변환 결과는 담당자가 확인 후 확정 수주 담당자 향후 도입 로드맵 항목

※ 처음부터 완벽할 필요는 없습니다. 떠오르는 항목부터 채우시고, 빈 칸은 상담에서 함께 정리해 드립니다. · 표준 기능으로 가능한지, 커스터마이징이 필요한지는 SyncLife가 검토해 알려드리니 고객사는 "무엇이 필요한지"만 적으시면 됩니다. · 구분은 아래 「점검하면 좋은 기능 목록」을 참고하세요.
